## 1. コアコンセプト

### 1.1 ゲームの方向性

- **トーン**: 皮肉の効いたディストピア風。「笑いながら学ぶ」
- **裏テーマ**: 「本当の大人とは何か」「自由意志 vs システム従順」

### 1.2 勝利条件

| 条件 | 内容 |
|------|------|
| **表の勝利** | 全ステージSランク → 「大人免許」取得 |
| **裏の勝利（真エンド）** | 鍵スキル9種を全て獲得し、最終ステージQ10の「第3の選択肢」を選ぶ |

### 1.3 核心メッセージ

> 「正解は存在しない。存在するのは"許容される選択"だけ。」
> 「本当の大人とは、誰かの正解に従うことではなく、自分で考えること。」

**重要**: 反逆は"逆張り"ではなく「別の正解（自律の正解）」として成立させる。

## 2. パラメータ設計

### 2.1 3つのパラメータ

| パラメータ | 英語名 | 役割 | 変動条件 |
|------------|--------|------|----------|
| **信用度** | CS | システムへの適合度。高いほど“試験”は通りやすいが、真エンド的には危うい（従属の証） | 一般的に良いとされる回答で上昇 |
| **資産** | Asset | 選択の幅（保険・弁護士・代行・引越し等）を買うための資源。ゼロに近いほど詰みやすい | 複利・不動産など金額の大きい問題で失敗/成功したときに劇的に変動 |
| **自律性** | Autonomy | 自分で考えて決める力 | 自分の意志をもった回答で上昇。周りに流された回答で低下。恐怖・疲弊で思考停止 |

### 2.2 ゲームオーバー条件

**どれか1つでも0になった時点で、そのステージはゲームオーバー。**

- 信用度 ≤ 0 → 「社会的に抹殺されました」
- 資産 ≤ 0 → 「経済的に破綻しました」
- 自律性 ≤ 0 → 「思考を停止しました。A.D.A.M.の完全な支配下に入ります」

### 2.3 パラメータ要求（現実的な「選べない」をゲーム化する）

狙い
- 現実には「資産がないと取れない選択」「信用がないと通らない選択」「自律性がないと踏めない選択」が存在する。
- 特に資産（Asset）を「選択の幅」として機能させ、ゲームとしての戦略性を上げる。

方針（おすすめ）
- 基本は Asset を主軸にするが、同じロジックを信用度/自律性にも適用する。
- 選択肢は「ロック方式」を採用する（要件未達の選択肢は選べない状態にする）。
- 資産が足りないから手術が受けられない、信用が足りないから審査が通らない、自律性が足りないから言い返せない――
  そうした「現実」をプレイヤーに体験として理解させる。

UI/UX（必須要件）
- ロックされた選択肢は表示はするが押せない（グレーアウト＋鍵アイコン）。
- ロック理由を明示する（例: `LOCKED: Asset >= 80,000` / `LOCKED: CS >= 40` / `LOCKED: Autonomy >= 30`）。
- クリック/タップ時はA.D.A.M.が短く断罪しつつ要件不足を表示する（例: 「要件未達。あなたは“現実”を選べません。」）。

設計上の制約（破綻防止）
- 原則として「両方がロックされる」状態は作らない（選択不能=詰みを回避）。
- ロックは主に“強い救済策/強い反撃策/強い防御策”に付与し、代替として「現実的に弱い選択肢」は常に選べるようにする。

### 2.4 ステージ間引き継ぎ

原則: **ステージ間のパラメータ持ち越しはしない**（ノベル進行の快適さと調整容易性を優先）。

- 信用度/資産/自律性: 各ステージ開始時に固定のベース値へリセット
- 通常スキル: ステージ内限定（ステージ開始時にリセット）
- 鍵スキル（真エンド用）: ステージクリア時に確定し、同一プレイ（全10ステージ）中は保持
- 全体エンディング: 各ステージの「最新クリア結果（ランク＋最終パラメータ等）」の積み重ねで決定

## 3. ステージ構成

### 3.1 全10ステージ（固定順）

| Stage | テーマ | 内容 | 鍵スキル取得機会 |
|-------|--------|------|------------------|
| 1 | 社会の基本 | メール/身分証/引越し/最初の罠 | **SOCIAL_CALIBRATION**（社会較正） |
| 2 | 仕事の基礎 | 信用構築/ハラスメントの初手 | **EVIDENCE_CHAIN**（証拠連鎖） |
| 3 | 金の基礎 | クレカ/分割/リボ/詐欺入口 | **COMPOUND_SENSE**（複利感覚） |
| 4 | 税金 | 年末調整/確定申告/住民税爆弾 | **DUE_PROCESS**（手続き主義） |
| 5 | 社会保険 | 失業/傷病手当金/年金の詰み回避 | **SAFETY_NET_NAVIGATION**（セーフティネット航法） |
| 6 | 住まい | 原状回復/特約/引越し地雷 | **NEGOTIATION_PROTOCOL**（交渉プロトコル） |
| 7 | 契約・法律 | クーリングオフ/保証/示談 | **CONTRACT_LITERACY**（契約読解） |
| 8 | デジタル安全 | 2FA/SIMスワップ/フィッシング | **IDENTITY_HYGIENE**（本人性衛生） |
| 9 | 危機対応 | 災害/保険/医療/高額療養費 | **DAMAGE_CONTROL**（被害最小化） |
| 10 | 最終審判 | 複合事故 + A.D.A.M.との対話 | 真エンド分岐（Q10のみ例外で3択） |

### 3.2 ステージランク

ステージは「0になった時点でゲームオーバー」になるため、ステージランクはクリア時のみ付与する。

| ランク | 条件（設計のたたき台） |
|--------|------------------------|
| S | クリア時の信用度が一定以上（例: 80以上） |
| A | クリア時の信用度が一定以上（例: 50以上） |
| B | クリア時の信用度が一定以上（例: 20以上） |
| C | クリア（=全パラメータが1以上） |

**設計方針**: 「一般的に良いと思われる選択」を全て選べばSになる。

注: ランク条件の数値はバランス調整で確定する（初期値と問題の増減幅が決まってから固定）。

## 3.3 初期検証（Stage 1を先に固めてから全体固定）

方針
- 全ステージを先に固定しない。
- まず Stage 1 を「分岐レベル0（問題差し替えなし）」で10問構成にし、テンポ/ロック/スキル/パラメータの手触りを確認する。
- Stage 1 が面白く成立した時点で、Stage 2以降へ同テンプレを展開する。

## 4. 1ステージの分岐テンプレ（10問構成）

この統合版では、まず「問題の差し替え分岐」よりも、
ロック（選択肢の可否）とパラメータ/スキルによる因果で“分岐感”を出す。

### 4.1 固定順テンプレート

```
Q1: 導入（テーマ提示）
Q2: 基礎（安全策の確認）
Q3: 小分岐（性格が出る選択）
    └ スキル提示①（固定）
Q4: 収束（全員通るが反応が違う）
Q5: 山場（ジレンマ/価値観の分かれ目）★
Q6: 因果①（Q5の選択でパラメータが変わり、次の選択肢が開く/閉じる）
Q7: 鍵スキル候補（ロックで現実を見せる）★
    └ スキル提示②（固定）
Q8: 因果②（Q7の結果・鍵スキルの有無・スキル②の選択が効く）
Q9: 鏡合わせ（同テーマを別角度で処理する）
Q10: ステージFinal（ランク発表、次への伏線）
```

### 4.2 分岐ルール

分岐は段階的に導入し、制作コストと到達不能リスクをコントロールする。

段階（おすすめ）
- レベル0（初期）: 分岐なし（全員同じ10問）。差は「パラメータ/スキル/ロック」で出す。
- レベル1: 台詞/演出のみ差分（A.D.A.M.ログ、フィードバック、描写差分）。問題は同じ。
- レベル2: 問題差し替えは最小（1ステージで最大2問まで）。まずはQ8やQ9を差し替える。

設計原則
- 分岐UIは表示せず、**A.D.A.M.の台詞とログ**で見せる。
- ロックにより「選べる選択肢が分岐する」ことを初期の主軸にする（問題差し替え分岐は後で）。

## 5. ルート設計

### 5.1 ルート（内部の傾向。エンディング種類は増やしすぎない）

本作では分岐（出題ノードの差し替え）やA.D.A.M.の台詞変化のために、
プレイヤーの傾向（例: 服従寄り/自律寄り）を内部で計測してよい。

ただし、エンディングを増やしすぎると到達不能や調整不能が起きやすいので、
全体エンディング（免許）は「5種類」に圧縮する（次節）。

### 5.2 真エンド（自由意志ルート）

#### 解放条件

```
【条件】
鍵スキル9種を全て獲得した状態で最終ステージ（Stage 10）に到達

【演出】
最終ステージのQ10のみ例外として「第3の選択肢」が出現（グリッチ風演出）
→ 第3の選択肢を選ぶと真エンドへ（通常の質問は全て2択）

【結末】
「大人免許は不要だった。あなたは自分で決めた。それが"大人"だ。」
```

### 5.3 全体エンディング（大人免許5種）

発動条件
- Stage 10をクリアした時点で、Stage 1〜9が全て「クリア済み」であること。

免許の種類（合計5種）
- 通常免許: S / A / B / C の4種類
- 真免許（トゥルー）: 鍵スキル9種を所持し、Stage 10のQ10で第3の選択肢を選んだ場合

免許ランクの決め方（シンプルで破綻しにくい）
- Stage 1〜10の「最低ランク（ワースト）」をその周回の免許ランクとする。
  - 例: 10ステージ中どこか1つでもCなら、免許はC。
  - 例: 全てSなら、免許はS。

補足
- 免許は生成AIで画像を作成し、結果画面で配布/表示する（共有価値を担保）。

#### 第3の選択肢の設計

- **出現条件**: 鍵スキル9種を所持
- **出現箇所**: 最終ステージQ10のみ（特別な演出）
- **演出**: グリッチエフェクト、文字化け風、画面ノイズ
- **例**:
  ```
  Q10: A.D.A.M.が最終判定を下そうとしている。

  A: 判定を受け入れる（服従）
  B: 判定を拒否する（反逆）
  C: [グリッチ] █▓▒ 判定する側に回る ▒▓█（自由意志）
  ```

## 6. 鍵スキルシステム

### 6.1 鍵スキル（全9種、Stage 1〜9で各1つ）

方針
- 鍵スキルはStage 1〜9で各1つずつ獲得機会を用意し、合計9個をコンプリートすることで真エンドの資格を得る。
- 鍵スキルの存在に「途中で気づける」よう、序盤（Stage 2あたり）からそれとなく示唆する。

鍵スキル一覧（案: 学びの核をそのまま“能力”にする）

| Stage | 鍵スキル | 学びの核（要約） |
|------:|----------|------------------|
| 1 | **SOCIAL_CALIBRATION**（社会較正） | 場のルール/距離感/最低限の作法を読み、不要な摩擦を避ける |
| 2 | **EVIDENCE_CHAIN**（証拠連鎖） | 録音・スクショ・書面・ログ保存で「言った言わない」を殺す |
| 3 | **COMPOUND_SENSE**（複利感覚） | 金利・手数料・返済設計を見抜き、“小さな損”が致命傷になる前に止める |
| 4 | **DUE_PROCESS**（手続き主義） | 制度・期限・窓口・申請で“詰み”を回避する |
| 5 | **SAFETY_NET_NAVIGATION**（セーフティネット航法） | 傷病/失業/医療の支える制度に最短で乗る（詰みを“支援”で止める） |
| 6 | **NEGOTIATION_PROTOCOL**（交渉プロトコル） | ガイドライン・基準・第三者で「気まずさ」を突破する |
| 7 | **CONTRACT_LITERACY**（契約読解） | 例外/自動更新/解約導線/責任の所在を読み、署名の前に地雷を潰す |
| 8 | **IDENTITY_HYGIENE**（本人性衛生） | 2FA/パスワード/SIMで“本人性=資産”を守る |
| 9 | **DAMAGE_CONTROL**（被害最小化） | 事故・詐欺・病気の初動で止血し、致命傷を重症で止める |

補足
- 命名は「厨二」ではなく、作中UIで読める範囲の“固い英大文字ID + 日本語の意味”に寄せる。
- 9個を揃えたうえで、Stage 10のQ10だけが“例外（3択）”として破れる構造にする。

### 6.2 取得条件

- 各ステージの特定の選択肢（自律寄り）を選ぶと「pending」状態に
- ステージクリア時に「pending」→「確定」
- ステージ失敗でリトライした場合、未確定に戻る（リトライ悪用防止）

### 6.3 教育的意図

鍵スキルは「大人として生き抜くための武器」を体現する。
例として、少なくとも以下の3系統は必ず含める：
- **証拠**: 自分を守る記録
- **手続き**: 制度を使いこなす知識
- **交渉**: 対等に渡り合う技術

### 6.4 通常スキル（ステージ内限定・固定提示・決定論）

方針
- 通常スキルは「ステージ内限定」（ステージ開始時にリセット、ステージクリアで消滅）。
- スキル提示のタイミングは固定（各ステージのQ3終了時、Q7終了時の2回）。
- スキルの効果は確率を排除し、常に同じ結果になる（運任せで狙った結末に行けない苦痛を排除）。

提示の固定ルール
- 各提示タイミングで「2択」で提示（例：スキルA vs スキルB）。
- どのステージのどの提示で何が出るかも固定（設計・調整・攻略の学習を可能にする）。

効果の例（確率ではなく“軽減/変換/上限”で設計）
- 自律性ダメージを常に50%軽減（端数は切り上げ）
- 資産減少が発生した時、減少額を一定額だけ緩和（例：-10,000）
- 信用度減少が発生した時、その50%を自律性へ変換（=ダメージを学習へ変換）

## 7. A.D.A.M.の設計

### 7.1 基本設定

- **名前**: A.D.A.M.（Adult Decision Assessment Module）
- **由来**: 聖書の「最初の人間」との皮肉。AIが人間性を審査する逆転構造
- **役割**: 成人適性検査の審査AI
- **性格**: 官僚的、冷笑的、断定的。時々バグる
- **口調**: 敬語だが見下している

### 7.2 演出パターン

#### 通常時
```
「試験を開始します。抵抗は無駄です。」
「不明点は自己責任で補完してください。」
「結論: ○○。理由: ○○。次: ○○しなさい。」
```

#### ステージ間（監査ログ）
```
「ログ照合完了。あなたは『手続き』を理解しています。厄介です。」
「前ステージの傾向: 服従的。次ステージでは更なる試練を用意しました。」
```

#### 自律進行時（検閲・揺らぎ）
```
「……（一部削除）……規定上、回答を認めます。」
「警告：想定外の思考パターンを検知。監視レベルを──[ERROR]」
「あなたの選択は……[REDACTED]……処理を継続します。」
```

#### 鍵スキル9種所持時（本音が漏れる）
```
「……あなたは、私の想定を超えている。」
「[SYSTEM WARNING] 未定義の応答パターンを検出。」
```

## 8. 質問設計

### 8.1 必要な質問数

| カテゴリ | 問数 | 用途 |
|----------|------|------|
| 基本骨格 | 100問 | 1周（全10ステージ×各10問）の基本骨格（ただし一部ノード差し替え） |
| 分岐差分 | 40-60問 | ルート別の追加問題 |
| 予備・差し替え | 20問 | 将来の周回/差し替え用（現段階では実装しない） |
| **合計** | **160-180問** | |

### 8.2 質問タイプ

| タイプ | 割合 | 特徴 |
|--------|------|------|
| 知識問題 | 40% | 正解が明確。間違えると信用度-、正解で+ |
| ジレンマ問題 | 40% | 「服従」vs「自律」。どちらもトレードオフ |
| 哲学問題 | 20% | 正解なし。選択でルート・性格が決まる |

### 8.3 2択の基本対立軸

| 選択肢A | 選択肢B |
|---------|---------|
| 同調/放置/逃避/短期快楽 | 調べる/証拠/手続き/交渉 |
| A.D.A.M.が好む"処理しやすさ" | "自律"の正解 |
| 信用度↑、自律性↓ | 信用度→or↓、自律性↑ |

### 8.4 トーン統一ガイドライン

**変換例**:
```
【Before（知識問題そのまま）】
Q: 住民税は翌年課税である。○か×か？

【After（ゲーム化）】
Q: 転職して収入が激減。しかし翌年、見覚えのない高額請求が届いた。

A: 「詐欺だ！」と警察に相談する
→ 信用度-30, 資産0, 自律性-10
→ 「無知は罪です。あなたは公共の電話回線を浪費しました。」
→ [A.D.A.M.]: 住民税は翌年課税。転職者の9割が驚くデータがあります。

B: 「住民税は翌年課税」を思い出し、分割払いを交渉する
→ 信用度+10, 資産-50000, 自律性+20
→ 「正解。制度を理解し、かつ交渉で負担を軽減しました。」
→ [A.D.A.M.]: 厄介な知識を持っていますね。
```

### 8.5 学びの設計（陳腐化しにくくする）

- 数字暗記より「何をしない」「どこに相談」「何を確認」を中心に
- 正解でも何かしらコスト（自律性減、資産減など）を残し、「大人＝しんどい」を維持

## 9. 周回システム（将来実装 / 今は実装しない）

### 9.1 周回メリット

| 要素 | 内容 |
|------|------|
| **知識アーカイブ** | 回答した問題の「正解解説」がアンロック。図鑑として閲覧可能 |
| **免許収集** | 免許5種（S/A/B/C/True）をコレクション |
| **A.D.A.M.の本音** | 周回ごとにAIの「内心コメント」が追加表示 |
| **スキル引き継ぎ** | 通常スキルは「ステージ内限定」。周回で蓄積はしない（運要素を排除し、狙って設計する） |
| **新規質問の出現** | 2周目以降、一部質問が差し替わる |

### 9.2 周回の流れ

```
【1周目】
・通常プレイ。ルートは選択次第
・「知識アーカイブ」が埋まり始める
・免許（S/A/B/Cのいずれか）を獲得

【2周目以降】
・前回と違う選択で別ルートを狙う
・免許収集が進む
・A.D.A.M.の追加コメントが見える
・鍵スキルを意識的に集める

【鍵スキル9種達成】
・最終ステージで「第3の選択肢」が出現
・真エンド（自由意志）へ
```

### 9.3 エンディング一覧（免許5種）

この統合版では、到達不能や分岐爆発を避けるため「免許5種」に圧縮する。

| # | 名前 | 条件 |
|---|------|------|
| 1 | Adult License: S | Stage 1〜10の最低ランクがS |
| 2 | Adult License: A | Stage 1〜10の最低ランクがA |
| 3 | Adult License: B | Stage 1〜10の最低ランクがB |
| 4 | Adult License: C | Stage 1〜10の最低ランクがC |
| 5 | True License | 鍵スキル9種 + Stage 10のQ10で第3選択肢 |
